## C++常见面试问题汇总

* 指针和引用的区别
	1. 使用指针的原因有：
		+ 指针能够有效地表示数据结构
		+ 能够动态分配内存，实现内存的管理
		+ 高效便捷使用数组和字符串
		+ 指针直接与数据的存储地址相关。
	
	2. 指针与引用的区别：引用本质上市别名，指针是地址; 指针是一个对象，引用是对象的别名
		+ 指针可以在运行时可以改变所指向的值（const 指针除外），而引用不可言
		+ 系统会为指针变量分配内存， 引用声明需要初始化，指向一个已经存在的对象，不能是空值。
		+ 在编译时候，编译器将指针和应用添加到符号表上，符号表上记录的是变量名以及变量对应的地址。符号表生成后不能更改，所以引用不可改变
		+ 指针的级数没有限制， 引用只可以有一级 
		+ int * const 是常量指针，指针本身不可变; int const * 是指向常量的指针，其指向的值不可变。

*  堆和栈的区别
	1. 堆资源由程序员管理，通过malloc/free,new/delete。容易产生内存泄漏，栈资源由编译器管理
	2. 对于堆，系用有记录空闲内存地址的链表，当系统收到程序申请，遍历该链表寻找大于所需要空间的堆节点，将该节点分配给程序。对于栈，只要栈的剩余空间大于所申请的空间，系统就会分配内存，否则就是栈溢出错误。
	3. 堆是不连续的内存区域; 栈市一块连续的内存区域，大小是操作系统预定的，linux通过ulimit -s指令查看大小 
	4. 堆在频繁的申请和释放操作中会造成大量的内存碎片，会降低程序效率。
	5. 生长方向：堆向高地址方向生长，栈向底地址方向生长。
	6. 堆动态分配，栈是动态分配和静态分配，由编译器完成。

* new和delete是如何实现的，new 与 malloc的异同处
	1. C++是通过new运算符进行动态内存申请，是基于类型进行的。
	2. new和delete是运算符，operator new/delete底层使用malloc和free实现。
	3. new的调用过程：new -> operator new() -> constructor function -> return ptr
	4. delete的调用过程：delete -> destructor function -> free()

* 在C++中const和static的用法（定义，用途）
	1. static可以用来隐藏变量或者对象的作用域，并且改变存储方式;
	2. 静态成员函数只能访问静态成员变量;
	3. const的全局变量是内部链接性，如需要改变其为外部链接性，则加上extern
	4. 函数也具有链接性，默认情况下其链接性为静态、外部的。也可以加上static关键字使其为内部链接性。
	
* const和static在类中使用的注意事项（定义、初始化和使用）
	1. const 常量：定义时就初始化，以后不能更改
	2. onst 形参：func(const int a){};该形参在函数里不能改变
	3. const修饰类成员函数：该函数对成员变量只能进行只读操作
	4. 只有const类对象能调用声明为const的成员函数
	5. const成员函数可以被对应的具有相同形参列表的非const成员函数重载

* C++中的const类成员函数（用法和意义）
	用法有：
	1. void fun(int a，int b) const{}
	2. void const fun(int a,int b){}
	这两种写法的本质是：void fun (const 类 *this, int a,int b);
	onst对象只能调用const成员函数,必须要提供一个const版本的成员函数
